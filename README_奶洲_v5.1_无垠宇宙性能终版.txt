《奶洲：五章双人撤离行动》v5.1
无垠宇宙 · 近距唤醒与自适应性能终版

一、使用方式

1. 双击 naizhou_v5_1_infinite_cosmos_final.html。
2. 建议使用最新版 Chrome、Edge、Firefox 或 Safari。
3. 游戏为单文件离线版，角色图片、第五章怪物原画和 160 种宇宙回收物均已内嵌，不需要联网。
4. 新版继续使用原来的 naizhou_save_v1 浏览器本地存档键，奶票、Lv.40 改造、角色选择、累计背包和图鉴记录保持兼容。

二、本次没有删减的内容

第一至第五章、十名奶聪、单双人同屏、Lv.40 共用升级、累计背包、怪物图鉴、章节专属掉落、神圣回收物、十五宗修仙大陆、无垠宇宙 16 领地、16 小兵、16 精英、16 领主、3 名宇宙级主宰以及 160 种第五章专属回收物全部保留。

三、第五章攻击逻辑调整

怪物不再隔着整张 16000×12000 地图锁定玩家。只有进入警戒范围、主动攻击怪物，或惊动同一小队时，敌人才会进入战斗。

基础警戒距离：
- 普通小兵：540
- 精英怪：690
- 星域领主：880
- 宇宙级主宰：1040

脱战距离采用更大的迟滞范围，避免怪物在边缘反复开战：
- 普通小兵：860
- 精英怪：1050
- 星域领主：1320
- 宇宙级主宰：1580

其他规则：
- 第五章开局前 6 秒为安全观察期。
- 通过乳星跃迁门后获得 1.35 秒短暂观察期。
- 远离战场后，怪物在 4.5—7.5 秒内脱离战斗并恢复休眠。
- 被玩家击中的怪物会立即反击，但只会唤醒附近的小规模同域队伍，不会拉响全图警报。
- 三名宇宙级主宰达到解锁条件后只会解除封印；玩家真正靠近王庭后才会开战。
- 攻击事件采用节流调度，避免多个敌人在同一帧同时释放整套弹幕。

四、性能优化

1. 视口裁剪
只绘制镜头附近的地块、建筑、容器、障碍、怪物、投射物、粒子和文字。地图另一端的对象不会再进入当前帧的绘制队列。

2. 远距离敌人休眠
未警戒的远处敌人停止寻路、瞄准、施法和复杂状态更新，只进行低频距离检查。进入警戒范围后才恢复完整 AI。

3. 星空分层缓存
第五章的星云和暗星被预渲染为 512×512 世界分块，并通过 LRU 缓存复用；明亮星体作为单独闪烁层绘制。省电、均衡、精细模式分别保留 24、36、52 个缓存分块，避免镜头移动时反复重绘星空。

4. 空间碰撞索引
玩家子弹不再逐颗检查全部 179 名初始敌人。第五章使用 360 像素网格索引，只检测投射物所在及相邻网格内的目标。

5. 投射物与特效动态上限
系统根据设备性能调整敌方投射物、危害区域、轨迹、粒子、飘字和技能特效数量。优先保留首领光束、横扫和高伤害预警，先清理过期或远离玩家的普通效果。

6. 远距离危害轻量更新
远离玩家的危害只继续倒计时并自然消失，不再进行碰撞判断、屏幕闪烁或镜头震动。

7. 小地图缓存
地图静态底图只绘制一次；动态标记按约 5—13Hz 更新，不再每帧重新绘制完整第五章。

8. 倒流历史降采样
时间回溯轨迹由每帧记录调整为 12Hz，视觉和判定精度仍足够，但显著减少长局中的数组更新。

五、自适应性能模式

游戏会结合触屏设备、CPU 核心数、可用内存和实际帧时间自动选择：
- 精细：高性能设备，保留最多粒子、轨迹和原始高阶危害绘制。
- 均衡：中等设备，适度减少阴影、轨迹和同屏攻击事件。
- 省电：移动设备或持续低帧率环境，继续使用真实怪物裁切主体，但减少昂贵模糊和多层叠加。

帧率持续下降时会自动降档；稳定一段时间后可恢复到设备允许的最高档位。第五章 HUD 会显示活跃/休眠怪物数量、当前性能模式和估算帧率。

六、验证结果

在资源中性测试构建中完成：
- 第五章正常生成 16000×12000 地图和 179 名初始敌人。
- 出生区域等待超过 7 秒，活跃敌人为 0，敌方子弹和危害均为 0。
- 接近领地后仅邻近 3—6 名敌人进入战斗，投射物和危害数量保持在自适应上限内。
- 撤回银河乳跃迁中枢后，敌人全部脱战休眠，残余子弹和危害归零。
- 连续开启两局第五章时，179 名敌人的 ID 唯一，星空分块与小地图缓存能够重新生成。
- 双人第五章可生成两名玩家并继续共享同一行动背包。
- 第一章回归启动尺寸仍为 2700×1750，未被第五章性能脚本影响。
- 八个脚本块全部通过 JavaScript 语法检查；23 个内嵌图像数据均可完整解码。
- 测试过程中未发现页面脚本异常或控制台错误。
